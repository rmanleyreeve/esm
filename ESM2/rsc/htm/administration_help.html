Administration

The Administration pages contain functions for the administration of the ESM system - adding and modifying users, comments, vessel details, exporting data etc.

Vessel Details

To change details about your vessel/installation simply click on the ‘Edit Vessel Details’ button.
This will bring up a box exactly as was used when setting up the system, vessel data, as well as the name and IMO reference and a category that best represents your vessel/installation. After any changes have been performed click ‘Submit’.

Manage Users

Users are the people you would like to give access to the system. What they are able to do within it can be controlled at any time. To add, edit or delete users go to the ‘Manage Users’ area.

To add a new user to the system click on the ‘Add User’ button.

This will open the following window:

This is a similar process to creating the administrator on set up.

Username:                  A username for quick reference.
Password:                   A secure password assigned by you.
Forename:                  The user’s first name or given name.
Surname:                    The user’s last name or family name.
Rank/Job Title:            Your official rank or job title.
Work ID:                      A unique identifier e.g. the seaman’s identity card or works payroll number.
Date of Birth:               The user’s date of birth by day/month/year.
Access Level:             The permissions the user has within the system.

Access level can be set by clicking on the ‘Access Level’ drop down list:

There are 5 possible settings:

  • Disabled:  User is not able to use the system; their information will be retained in the system and can be reactivated if required.
  • Moderated User: User has ‘read only’ access to the system. They can post comments but their comments must be approved before they display.
  • Approved User: User has ‘read and write’ access, can add/edit/delete other users, can create space entries and perform audits. Approved users can post comments and upload photos and documents without needing to be moderated (i.e. they will appear immediately) and also approve the comments of moderated users. They cannot sign off a space as complete.
  • User with Sign-Off: User has all above permissions and in addition is able to verify spaces as complete. This user does not have access to the administration panel.
  • Administrator:  Complete access to all areas and permissions.­

Existing users can also be modified by clicking the ‘Managing Existing User’ list here:
 

Selecting a user via the drop down list:

And with the desired user selected click ‘Edit’.

To remove the selected user click ‘Delete’.
To change the selected user's password click ‘Change Password’.
Clicking ‘Change Password’ will bring up the following box:

Simply enter the new password into the field labelled ‘New Password’. Then write the new password again exactly as in the first field in the ‘Confirm Password’ field to confirm.
This rests the password for that user and their previously held password will no longer work.
You will need to inform them of their new password.

Manage Comments

Users with the required permissions are able to approve the comments of moderated users.
Approving the comment means it can be seen by all system users, so should only be approved if the user believes them to be correct and a valid contribution.
In addition a user’s comment maybe edited or deleted.

N.B if you decide not to approve a user’s comment we advise that you communicate to them why the comment was not approved, so they are able to learn from the experience and make a more valuable contribution in the future.

To approve a comment select it from the list and click the ‘View Comment’ button.

This will then take you to the appropriate space and you will see the unapproved comment in red:

The comment may then be edited, deleted or approved by clicking the appropriate button.

Data Export Functions

 

Database Backup

It is important that you make regular backups of your ESM data. This ensures that, should anything happen to your computer, the data you have collected is not lost. To create a backup click the ‘Data Export File’ button. Once you have done so you should see the following message:

Clicking ‘OK’ will reveal a system window showing you where the backup file has been stored.

You should copy the file to a safe location.

 

Data Export File

To be able to share the information you collect with your office it needs to be exported. This can be done using an Internet connection or be generated as a Zip file which can then be sent to the web system by email.

Whichever you choose, you will have the option of sending ‘All Data’ or to ‘Exclude Photos & Documents’.

This is designed to enable you to control the size of the export. Exporting without photos and documents will generate a much smaller file. You should check with your IT department to see if there are any restrictions on bandwidth and email attachments.

 

Sending via email

To generate a file to email, click the option ‘Data Export File’

You should then receive a message stating ‘Data Export file created successfully’.

Clicking ‘OK’ will generate a system window showing you where the backup file has been stored.
You should then email the zip file to export@esm-system.com

 

Sending via the Internet

To send an export via the Internet, click the option ‘Send Data File’.

You should then receive a message stating ‘File created and uploaded successfully’.

Clicking ‘OK’ will generate a system window showing you where the backup file has been stored.
